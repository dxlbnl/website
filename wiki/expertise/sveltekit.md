# SvelteKit patterns

Reference for the patterns used in this project. Read this instead of `node_modules/`.
Official docs at `kit.svelte.dev`.

---

## Route file conventions

| File | Purpose |
|---|---|
| `+page.svelte` | Page component — rendered on both server and client |
| `+page.ts` | Universal load — runs on server at SSR time and on client during navigation |
| `+page.server.ts` | Server-only load — DB access, auth, cookies, private env vars |
| `+layout.svelte` | Layout wrapping all descendant routes in the directory |
| `+layout.ts` | Universal layout load — data available to layout + all descendants |
| `+layout.server.ts` | Server-only layout load — auth guards, shared server data |
| `+server.ts` | API route — handles HTTP verbs, returns `Response`, no page rendering |
| `+error.svelte` | Error boundary for the nearest layout scope |

**Route groups** `(name)/` — directory-only grouping, does not appear in the URL. Used in this project for `(console)/` (public, nav+footer layout) and `(private)/` (admin+invoices, auth-gated).

**`[slug]`** — dynamic segment. `params.slug` in the load function.

## Load functions

### `+page.server.ts` — server-only load

For any page that reads the DB, checks auth, or needs private env vars.

```ts
import type { PageServerLoad } from './$types'
import { error, redirect } from '@sveltejs/kit'

export const prerender = false  // always false for server loads

export const load: PageServerLoad = async ({ params, locals, cookies, fetch, parent }) => {
  const parentData = await parent()  // data from +layout.server.ts up the tree

  const item = await db.query.things.findFirst({ where: eq(things.slug, params.slug) })
  if (!item) throw error(404, 'Not found')

  return { item }  // becomes PageData — typed via ./$types
}
```

### `+page.ts` — universal load

For prerendered/static pages. Runs server-side at build time and client-side during navigation.

```ts
import type { PageLoad } from './$types'

export const prerender = true  // can be omitted if the layout already sets it

export const load: PageLoad = async ({ params, fetch, data, parent }) => {
  // `data` here is the data from a sibling +page.server.ts if one exists
  return { slug: params.slug }
}
```

### `+layout.server.ts` — auth guard / shared server data

Data returned here is available in the layout component and merged into every descendant page's `PageData`.

```ts
import type { LayoutServerLoad } from './$types'
import { redirect } from '@sveltejs/kit'

export const load: LayoutServerLoad = async ({ cookies, locals }) => {
  if (!verifyAdminSession(cookies.get('admin_session'))) {
    throw redirect(303, '/admin/')
  }
  return { user: locals.user }  // available as data.user on layout + all child pages
}
```

Descendants add **no** auth code — the layout guard applies automatically.

### Consuming data in a page

```svelte
<script lang="ts">
  import type { PageData } from './$types'
  let { data }: { data: PageData } = $props()
</script>

<h1>{data.item.title}</h1>
```

`$types` is generated by SvelteKit — never write it by hand. The type of `data` is always `PageData` for pages and `LayoutData` for layouts.

## API routes (`+server.ts`)

Handle HTTP verbs directly. Return a `Response` or use the SvelteKit helpers.

```ts
import { json, error } from '@sveltejs/kit'
import type { RequestHandler } from './$types'

export const POST: RequestHandler = async ({ request, cookies, locals }) => {
  const body = await request.json()
  if (!body.email) throw error(400, 'email required')

  await doSomething(body.email)
  return json({ ok: true })
}

export const GET: RequestHandler = async ({ url }) => {
  const id = url.searchParams.get('id')
  return new Response(JSON.stringify({ id }), {
    headers: { 'content-type': 'application/json' }
  })
}
```

`json(value, init?)` is a shorthand for `new Response(JSON.stringify(value), { headers: { 'content-type': 'application/json' }, ...init })`.

## Form actions

Defined in `+page.server.ts` alongside the load. Used for server-side form handling without JavaScript.

```ts
import type { Actions } from './$types'
import { fail, redirect } from '@sveltejs/kit'

export const actions: Actions = {
  default: async ({ request, cookies }) => {
    const data = await request.formData()
    const email = data.get('email')?.toString()
    if (!email) return fail(400, { email, missing: true })

    await subscribe(email)
    throw redirect(303, '/thanks/')
  },

  delete: async ({ request }) => {
    // named action — <form method="POST" action="?/delete">
  }
}
```

In the component, consume action result via `form` prop:

```svelte
<script lang="ts">
  import type { ActionData } from './$types'
  let { form }: { form: ActionData } = $props()
</script>

<form method="POST">
  <input name="email" />
  {#if form?.missing}<p>Email is required</p>{/if}
  <button>Subscribe</button>
</form>
```

## Hooks (`src/hooks.server.ts`)

The `handle` hook intercepts every request. Used in this project for palette injection, security headers, and pageview logging.

```ts
import type { Handle } from '@sveltejs/kit'

export const handle: Handle = async ({ event, resolve }) => {
  // Read cookies, set locals, modify request
  event.locals.user = getUserFromCookies(event.cookies)

  const response = await resolve(event, {
    transformPageChunk: ({ html }) => html.replace('<html', `<html data-foo="bar"`)
  })

  // Modify response headers
  response.headers.set('X-Custom', 'value')

  return response
}
```

`event.locals` is the place to stash per-request server-side state (e.g. authenticated user). It is accessible in all load functions and API routes as `locals`.

## `prerender` flag

| Value | When to use |
|---|---|
| `true` | Content pages built at build time from markdown (public `(console)/` routes) |
| `false` | Pages that read the DB at runtime, check auth, or call external APIs |
| `'auto'` | SvelteKit decides — avoid, be explicit |

Admin and API routes always need `prerender = false`. Setting it in `+layout.server.ts` applies it to all descendants.

## Redirects and errors

```ts
import { redirect, error, fail } from '@sveltejs/kit'

// In a load function or action:
throw redirect(303, '/target/')   // 303 = POST→GET redirect; use 307 to preserve method
throw error(404, 'Not found')     // caught by nearest +error.svelte
return fail(400, { field: 'bad' }) // action validation failure — does NOT throw
```

## Environment variables

### `$env/static/private` — private, build-time

Inlined at build time. Only available server-side. Use for API keys, tokens, secrets.

```ts
import { RESEND_API_KEY, DATABASE_URL, ADMIN_TOKEN } from '$env/static/private'
```

Values must be present at build time. Accessing them client-side is a build error.

### `$env/static/public` — public, build-time

Inlined at build time. Available on both server and client. Prefix: `PUBLIC_`.

```ts
import { PUBLIC_SITE_URL, PUBLIC_COUNTRIES_EU } from '$env/static/public'
```

### `$env/dynamic/private` — private, runtime

Read at request time from `process.env`. Server-side only. Use when the value changes between deploys without a rebuild (e.g. feature flags, runtime secrets on edge).

```ts
import { env } from '$env/dynamic/private'
const key = env.RESEND_API_KEY
```

### `$env/dynamic/public` — public, runtime

Read at request time. Available on both server and client. Prefix: `PUBLIC_`.

```ts
import { env } from '$env/dynamic/public'
const siteUrl = env.PUBLIC_SITE_URL
```

**This project uses `$env/static/*` throughout** — values are set at build time on Vercel. Prefer static over dynamic unless you specifically need runtime re-reads.

## `$app/*` imports

```ts
import { goto, invalidate, invalidateAll, beforeNavigate, afterNavigate } from '$app/navigation'
import { page } from '$app/state'          // reactive: page.url, page.params, page.data, page.status
import { navigating } from '$app/state'    // null when idle, NavigationTarget during navigation
import { resolve, base } from '$app/paths' // resolve('/path/') prepends base path
import { browser, dev, building } from '$app/environment' // runtime booleans
```

`page` from `$app/state` is Svelte 5's rune-based replacement for the old `$page` store. Access it directly: `page.url.pathname`, `page.params.slug`.

## Cookies API

```ts
// In any server load or API route:
const value = cookies.get('name')
cookies.set('name', 'value', {
  path: '/',
  httpOnly: true,
  secure: true,
  sameSite: 'lax',
  maxAge: 60 * 60 * 24 * 7  // 1 week in seconds
})
cookies.delete('name', { path: '/' })
```

Always set `path` explicitly — SvelteKit does not default to `/`.

## Svelte 5 runes

```svelte
<script lang="ts">
  // Props — replaces `export let`
  let { name, count = 0, onchange }: {
    name: string
    count?: number
    onchange?: (v: number) => void
  } = $props()

  // Bindable prop — parent can bind:value
  let { value = $bindable('') }: { value?: string } = $props()

  // Reactive state — replaces `let` with reactivity
  let x = $state(0)
  let items = $state<string[]>([])

  // Derived — replaces `$: computed`
  const doubled = $derived(x * 2)
  const filtered = $derived(items.filter(i => i.length > 3))

  // Complex derived
  const result = $derived.by(() => {
    return expensiveComputation(x)
  })

  // Side effect — replaces `$:` with side-effects and `onMount`
  $effect(() => {
    console.log('x changed to', x)
    return () => { /* cleanup */ }
  })

  // Inspect (dev only — logs when value changes)
  $inspect(x)
</script>
```

**No `export let`, no `$:`, no `onMount` for reactive state — use runes.**

### Event handlers (Svelte 5)

```svelte
<!-- Old Svelte 4 -->
<button on:click={handler}>Click</button>

<!-- Svelte 5 -->
<button onclick={handler}>Click</button>
<input oninput={(e) => (value = e.currentTarget.value)} />
```

### Snippets (replaces slots)

```svelte
<!-- Parent -->
<script>
  import Card from './Card.svelte'
</script>

{#snippet header()}
  <h2>My heading</h2>
{/snippet}

<Card {header} />

<!-- Card.svelte -->
<script lang="ts">
  import type { Snippet } from 'svelte'
  let { header, children }: { header?: Snippet; children: Snippet } = $props()
</script>

{@render header?.()}
{@render children()}
```
